In the Works: A Windows PDF Reader Built with PyMuPDF

Reading PDFs on Windows has become a daily routine—whether reviewing documents or reading guides. Unfortunately, most off-the-shelf PDF readers felt bloated or slow, and I wanted fine-grained control over rendering and annotation without pulling in dozens of megabytes of dependencies.

That’s why I decided to build a pure-Python PDF viewer using PyMuPDF (a.k.a. Fitz) for parsing/rendering, paired with PyQt5 for the UI layer. In this post, I’ll walk through why I chose this stack, how the application is structured, and lessons learned along the way—all focused on the Windows platform.

Key Features Overview

Fast Page Rendering (Windows-optimized). Uses PyMuPDF’s native C backbone to generate bitmaps quickly. Lazy-loads pages: only the current viewport (and optionally the next page) is rendered.

Lightweight & Python-Only

Pure Python code—no heavyweight Electron or .NET dependencies. Single executable under 15 MB (after packaging with PyInstaller).

Built with PyQt5 to match native Windows controls (File dialog, menus). Keyboard shortcuts familiar to Windows users: Ctrl+O for open, Ctrl+F for find, Ctrl++/Ctrl– to zoom, PageUp/PageDown to navigate pages.

One-click search bar builds an on-the-fly inverted index over all pages. Jumps directly to matching page and highlights occurrences.
